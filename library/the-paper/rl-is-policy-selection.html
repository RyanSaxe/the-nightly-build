The RL That Only Picks

A token-level audit finds that reinforcement learning for math reasoning rewrites only one to three percent of a model's choices, always to a word the base model already ranked near the top, and that a cheap trick reproduces the gain for a thousandth of the cost.

11 min read2026-07-14

Rethinking RL for LLM Reasoning: It's Sparse Policy Selection, Not Capability Learning

Ömer Faruk Akgül, Rajgopal Kannan, Willie Neiswanger, Viktor Prasanna · arXiv:2605.06241 · 2026

Read the paper
Reinforcement learning has become the standard for improving reasoning in large language models, yet evidence increasingly suggests that RL does not teach new strategies; it redistributes probability mass over solutions the base model already contains. In this work, we ask: if RL merely steers the model toward paths it already knows, is the RL optimization loop itself necessary? Through token-level analysis across multiple model families and RL algorithms, we find that RL's beneficial footprint is a sparse, predictable correction concentrated at high-entropy decision points where the model is uncertain which branch to take. Only 1–3% of token positions are affected, the promoted token always lies within the base model's top-5 alternatives, and targeted corrections at those few positions causally recover a large fraction of RL's accuracy gain, while random corrections fail. The base model's own entropy identifies these positions without any RL-trained model, and the entire correction is low-dimensional, representable in a tiny fraction of model parameters. These findings reframe reasoning improvement as sparse policy selection, not capability acquisition. We translate this insight into ReasonMaxxer, a minimal RL-free method that applies contrastive loss only at entropy-gated decision points, using a few hundred base-model rollouts and no online generation. Across three model families, six scales, and six math reasoning benchmarks, ReasonMaxxer matches or exceeds full RL performance while requiring only tens of problems and minutes of single-GPU training, a reduction in training cost of roughly three orders of magnitude.1

What everyone thinks RL is doing

The reigning recipe for a reasoning model looks like discovery. Take a base model, reward it when it reaches a verified answer, and let a policy-gradient method such as GRPO push probability toward whatever rollouts worked. DeepSeek-R1 popularized the loop, and the felt experience of watching the model grow longer chains of thought is that it is learning to reason.23 This paper argues the feeling is mostly wrong, and it does so by looking at the one place the question can be settled: the tokens.

It is not alone in its suspicion. A line of recent work has chipped at the discovery story from the outside. Yue and colleagues showed that reinforcement learning raises a model's single-sample accuracy while often lowering its pass@k, meaning the trained model succeeds more reliably but explores a narrower set of solutions than the base model already held.4 Others found that the reward signal barely matters: on Qwen-family models, even random or spurious rewards produce most of the reported gain, which only makes sense if the ability was already present and RL is eliciting rather than teaching it.5 Akgül and colleagues take that circumstantial case and make it mechanistic.

The edit is smaller than you think

The first experiment lines a base model up against its own RL-trained version and decodes the same problems greedily, comparing the two token by token. A disagreement can be one of two things. It can be a reranked token, where the RL model picks a word the base model also had near the top of its list, or a shifted token, where the RL model reaches for something the base model ranked far down or not at all. Discovery predicts shifted tokens. The paper finds almost none.

Decision point A token position where the base model's next-token distribution is high-entropy: it is genuinely unsure which of several continuations to take. These are the forks in a reasoning trace, and they turn out to be where nearly all of RL's edits land.

Across four base-to-RL pairs spanning Qwen2.5 at 1.5B and 7B, both GRPO and PPO, and Qwen3-4B, only about one to three percent of token positions change between the base and the trained model, the promoted token sits on average around rank two and always inside the base model's top five, and no genuinely new tokens are introduced.1 The positions that do change are not random: they carry roughly seven to twelve times the base model's entropy of the positions that stay fixed. RL, in other words, is not writing new reasoning. It is breaking ties the base model did not know how to break.

Proving those few tokens carry the gain

A correlation between changed tokens and high entropy is suggestive, not decisive. The paper earns the causal claim with a substitution test. Let the base model generate, but at exactly the reranked positions overwrite its choice with the token the RL model preferred, and leave every other position alone. If those few forks are where the benefit lives, this should reconstruct the RL model's accuracy. It does: the oracle substitution recovers the trained model's single-sample accuracy on every pair, while touching only one to four percent of tokens.1

The oracle is a diagnosis, not yet a treatment, because it still uses the trained model to say which token to write. But it converts the thesis from a description into something falsifiable and then survives the test. It also lands on the same object other groups arrived at independently: a small set of high-entropy "forking" tokens that a reasoning trace pivots on, and that carry most of the useful gradient during RL.6

The base model already knows where it is unsure

If the trained model is needed only to locate the forks, perhaps it is not needed at all. The next result removes it. Gating on the base model's own entropy, with no access to the RL model, flags the correctable positions well enough that correcting them matches the teacher on the 7B GRPO pair and improves the others substantially, touching between one and eight percent of tokens.1 Entropy alone answers the where. What remains is the which: at a flagged fork, which of the top few tokens to prefer. That the model's own uncertainty is a good map to its correctable mistakes echoes a separate finding that simply minimizing a reasoning model's entropy can raise its accuracy with no reward signal at all.7

The correction is also small in parameter terms. Distilling the teacher's behavior into a rank-32 adapter, touching under half a percent of the weights, reproduces its accuracy, which says the change RL makes is genuinely low-dimensional rather than a diffuse rewrite of the network.1

ReasonMaxxer: the same gain without the loop

The method that falls out of the diagnosis is deliberately small. It never runs the online RL loop and never needs a trained teacher.

  1. Sample the base model

    Take a few hundred rollouts from the base model on a small problem pool, and keep only problems it solves sometimes but not always, so there is a signal to learn from.1

  2. Mark the forks by entropy

    Flag the decision points where the base model's entropy exceeds a threshold. These, and only these, are where the method will act.

  3. Nudge at the forks, anchor elsewhere

    Apply an advantage-weighted contrastive loss at the flagged tokens, pushing toward the choices that led to correct answers, while a term that keeps the rest of the distribution close to the base model prevents collateral drift.

The result is a rank-32 adapter trained for a single epoch on the order of a thousand short sequences, chosen on a fifty-problem validation split. Across three model families, six scales, and six math benchmarks, it matches or beats full RL pipelines. On Qwen2.5-32B it reaches an average of 0.440 for about twenty-five dollars of compute against the 0.436 of a full RL run, Open-Reasoner-Zero, reported at roughly a hundred thousand dollars,8 the three-orders-of-magnitude gap the abstract claims.1

Where the claim stops, and where its title overreaches

The mechanistic core is the strong part, and it is strong because it is controlled: the reranked-versus-shifted split, the oracle substitution with its random-correction control, and the entropy locator triangulate on the same conclusion from three directions, and they agree with independent work on forking tokens and on pass@k narrowing.46 But the generalization the title asserts runs ahead of the evidence, and a careful reader should hold the two apart.

What holds up
  • The sparse, top-5, entropy-concentrated footprint of RL, shown and then causally confirmed.
  • A genuinely cheap recipe worth trying before you spend on GRPO for a verifiable-answer task.
What to be careful about
  • Seven of the eight bases are Qwen or Qwen-distilled, exactly the family where spurious rewards already worked; the one non-Qwen base barely reasons at all.5
  • Results are single-sample. The paper reports no pass@k, so whether its cheap method narrows the solution set the way RL does is untested on the axis where RL looks worst.4
  • Math only, with several baseline costs estimated rather than measured, so the headline savings are directional.
Verdict A well-controlled diagnosis of what outcome-reward RL does to a capable math reasoner, plus a cheap recipe that exploits it. Read the title with one silent correction: the evidence supports "outcome-reward RL on verifiable math with a strong base," not RL for reasoning in general. A pass@k table and one non-Qwen result off the floor would decide whether the thesis travels.1

Read it yourself

The reconstruction compresses two things worth seeing whole: the exact token-diff tables and the ReasonMaxxer objective.1

  • read The token-level analysis: the reranked-versus-shifted counts and the entropy ratios. This is the evidence the whole argument rests on, and it is legible in one table.
  • read The oracle-substitution result: note that random substitution of the same token count does nothing. The contrast is the causal claim.
  • skim The ReasonMaxxer results table: read the cost column next to the accuracy column, and find the one row where full RL still wins before you quote "matches or exceeds."
  • skip The per-benchmark appendices on a first pass; the mechanism, not the leaderboard, is the reason to read this paper.

If it survives a pass@k test and a non-Qwen base, it stops being a provocative result about Qwen math and becomes a general statement about what reinforcement learning is for. Either way, the token-level method is the part to keep: it is a cheap, repeatable way to ask any RL run what, exactly, it changed.
