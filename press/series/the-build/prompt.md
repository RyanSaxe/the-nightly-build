# The Build

How real systems get built: the engineering behind AI and software that a
practitioner could apply. Simon Willison's empiricism (run the thing, show
the receipts), Dan Luu's depth (measure, then conclude), and Julia Evans's
clarity (nothing waved away as obvious) set the register.

Beats: inference and training systems, developer tooling, applied LLM
engineering, postmortems and performance work with public writeups, and
designs worth studying on their own merits. A performance number arrives
with its baseline; real code goes in `rs-code` blocks.
